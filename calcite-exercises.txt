# 🔰 Phase 1: Core Muscle Memory (Don’t skip this)

Goal: Stop thinking about syntax.

### Practice:

* Build a simple dashboard using:

  * `<calcite-shell>`
  * `<calcite-navigation>`
  * `<calcite-panel>`
  * `<calcite-button>`
  * `<calcite-card>`

### Exercise:

👉 Recreate a **basic admin panel UI**

* Sidebar navigation
* Header bar
* Main content cards

### What to focus on:

* Slot usage (`slot="header"`, `slot="footer"`)
* Layout hierarchy (Shell → Panel → Content)

---

# ⚙️ Phase 2: Component Behavior (Where most people fail)

Goal: Understand interaction patterns, not just UI.

### Practice:

Use:

* `<calcite-modal>`
* `<calcite-dropdown>`
* `<calcite-action-bar>`
* `<calcite-alert>`

### Exercise:

👉 Build a **settings panel**

* Button → opens modal
* Modal → form inputs
* Save → triggers alert

### Key skill:

👉 Controlling components via JavaScript:

```js
document.querySelector("calcite-modal").open = true;
```

---

# 🗺️ Phase 3: Real Power — Map Integration

This is where Calcite becomes valuable.

### Practice:

Integrate with:

* ArcGIS Maps SDK for JavaScript

### Exercise:

👉 Build a **map app UI**

* Map on the right
* Calcite sidebar on the left

### Stretch:

* Add a `<calcite-panel>` that updates when clicking map features

# 💡 Pro Techniques (Most people never reach this)

### 1. Layout mastery

* Combine:

  * `calcite-shell-panel`
  * `calcite-splitter`
* Build resizable panels (very pro feel)

---

### 2. Event-driven UI

Listen to events like:

```js
element.addEventListener("calciteSwitchChange", (e) => {
  console.log(e.target.checked);
});
```

---

### 3. Theming

Override CSS variables:

```css
:root {
  --calcite-color-brand: #ff6600;
}
```

# 🚫 Common Mistakes (avoid these early)

* Treating Calcite like Bootstrap (it’s not)
* Ignoring slots → leads to broken layouts
* Not using panels → results in messy UI
* Overusing divs instead of Calcite structure
